## 📢 네트워크 통신 방식의 다양한 종류 

네트워크 통신은 데이터를 주고받는 방식과 대상의 수에 따라 여러 가지 방식으로 나뉩니다. 유니캐스트(Unicast) 외에 주요 통신 방식인 **브로드캐스트**, **멀티캐스트**, 그리고 **애니캐스트**를 설명합니다.

---

### 1. 👥 유니캐스트 (Unicast)

가장 흔한 통신 방식으로, **1:1 통신**을 의미합니다.

* **설명**: 출발지에서 지정된 **하나의 목적지**로만 데이터를 전송합니다.
* **특징**: 대부분의 인터넷 트래픽(웹 서핑, 이메일, 파일 전송)이 유니캐스트 방식입니다. 수신을 원하지 않는 장치는 데이터를 처리하지 않습니다.
* **주소**: 출발지 및 목적지 **MAC 주소**와 **IP 주소**가 모두 특정 장치(Host)의 주소로 지정됩니다.

---

### 2. 🌍 브로드캐스트 (Broadcast)

네트워크에 연결된 **모든 장치**에게 데이터를 한 번에 전송하는 방식입니다.

* **설명**: 출발지에서 동일 네트워크(서브넷) 내의 **모든 호스트**에게 데이터를 전송합니다.
* **특징**: 수신을 원하든 원하지 않든, 네트워크상의 모든 장치가 해당 데이터를 수신하고 처리해야 합니다.
    * 라우터에 의해 **브로드캐스트 도메인**이 분리됩니다. 즉, 브로드캐스트 트래픽은 라우터를 넘어가지 못합니다.
* **주소**: 목적지 MAC 주소는 `FF:FF:FF:FF:FF:FF` (All F's)로, 목적지 IP 주소는 **브로드캐스트 주소**로 지정됩니다.
* **사용 예**: **ARP(Address Resolution Protocol)** 요청, DHCP(IP 주소 할당) 요청 등이 대표적입니다.

---

### 3. 📡 멀티캐스트 (Multicast)

특정 그룹에 속한 **다수의 장치**에게만 데이터를 효율적으로 전송하는 방식입니다. **1:N 통신**을 지원합니다.

* **설명**: 출발지에서 데이터를 전송하면, 해당 데이터를 수신하겠다고 미리 등록한 (**그룹에 가입한**) 장치들에게만 전송됩니다.
* **특징**: 브로드캐스트처럼 불필요한 장치에게 데이터를 보내지 않아 효율적이며, 유니캐스트처럼 수신자 수만큼 복사본을 만들 필요가 없어 대역폭을 절약합니다.
* **주소**: **멀티캐스트 IP 주소**(`224.0.0.0` ~ `239.255.255.255` 범위)와 이에 매핑되는 **멀티캐스트 MAC 주소**를 사용합니다.
* **사용 예**: IPTV (실시간 스트리밍), 화상 회의, 라우팅 프로토콜(OSPF, EIGRP)의 정보 교환.

---

### 4. 🔀 애니캐스트 (Anycast)

동일한 IP 주소를 가진 **가장 가까운(가장 빠른) 하나의 장치**로만 데이터를 전송하는 방식입니다.

* **설명**: 출발지에서 데이터를 전송하면, 동일한 IP 주소를 공유하는 여러 서버 중 **네트워크적으로 가장 가까운** 서버에게만 데이터가 전달됩니다. 이는 **1:1-of-N 통신**으로 간주됩니다.
* **특징**: 라우팅 프로토콜이 가장 효율적인 경로를 결정하여 트래픽을 분산시키고 부하를 줄입니다.
* **사용 예**: **DNS 루트 서버**나 **CDN(Content Delivery Network)** 서버와 같이 전 세계에 분산 배치된 서비스에 주로 사용됩니다. 사용자가 어느 위치에 있든 가장 가까운 서버로부터 빠르게 응답을 받을 수 있게 합니다.

---

### 📚 통신 방식별 비교 요약

| 통신 방식 | 전송 대상 | 목적지 주소 | 효율성 | 주요 사용처 |
| :---: | :---: | :---: | :---: | :--- |
| **유니캐스트** | 특정 **1개**의 장치 | 개별 MAC/IP 주소 | 높음 | 웹, 이메일, 파일 전송 |
| **브로드캐스트** | **동일 네트워크 내 모든** 장치 | 브로드캐스트 MAC/IP 주소 | 낮음 (네트워크 부하 유발) | ARP 요청, DHCP 요청 |
| **멀티캐스트** | **그룹에 가입된 다수**의 장치 | 멀티캐스트 MAC/IP 주소 | 매우 높음 | IPTV, 화상 회의, 라우팅 정보 교환 |
| **애니캐스트** | **가장 가까운 1개**의 장치 (N개 중) | 동일 IP 주소 | 매우 높음 | DNS 루트 서버, CDN |

이러한 통신 방식들은 네트워크 장비와 프로토콜에 의해 구별되어 처리되며, 특히 **멀티캐스트와 애니캐스트는 대규모 분산 환경**에서 서비스 품질과 효율성을 결정하는 중요한 요소입니다.
