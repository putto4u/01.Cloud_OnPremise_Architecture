## TCP/IP라는 명칭과 대표성

네트워크 통신을 설명할 때 흔히 기술적으로 **TCP/IP**라고 결합하여 부르는 이유가 있습니다. 
이는 단순히 두 프로토콜의 나열이 아니라, 현대 인터넷의 근간을 이루는 **'계층적 구조'**와 **'역할 분담'**을 상징하기 때문입니다.

---
### 1. 핵심 차이: "어떻게" vs "어디로"


| 구분 | IP (Internet Protocol) | TCP (Transmission Control Protocol) |
| --- | --- | --- |
| **주요 역할** | 주소 지정, 패킷 라우팅 (배송) | 데이터 흐름 제어, 에러 복구 (검수) |
| **단위** | 패킷 (Packet) | 세그먼트 (Segment) |
| **특징** | 비연결형, 신뢰성 보장 안 함 | 연결 지향형, 신뢰성 보장 |

* **IP (Internet Protocol):** 데이터를 목적지까지 전달하는 **'배송 주소'**와 **'경로(Routing)'**를 담당합니다.  
*   하드웨어적인 연결망을 넘어 전 세계 네트워크를 하나로 묶는 논리적 주소 체계입니다.  
* **TCP (Transmission Control Protocol):** IP 위에서 작동하며, 데이터가 유실되지 않았는지, 순서가 바뀌지 않았는지 확인하는 **'신뢰성 제어'**를 담당합니다.  
* TCP/IP는 특정 프로토콜 두 개만을 의미하는 것이 아니라(UDP 등), 인터넷 통신에 필요한 **프로토콜 집합체(Suite)**를 통칭하는 고유 명사입니다.  
* **표준화의 역사:** 1970년대 DARPA(미 국방성)에서 연구될 당시부터 이 두 프로토콜이 핵심 엔진 역할을 했습니다.  
*   이후 이 모델이 전 세계 표준이 되면서 '인터넷 = TCP/IP 기반 네트워크'라는 공식이 성립되었습니다.  
* **계층간 독립성:** TCP는 4계층(전송 계층), IP는 3계층(네트워크 계층)에 위치합니다. TCP 대신 UDP를 사용할 수도 있지만, 여전히 하부에는 IP가 필수적입니다.  
*   따라서 가장 대표적인 조합인 TCP/IP가 전체 모델을 상징하게 되었습니다.

### 4. 자주 오해하는 부분: UDP와의 관계

"IP 위에는 TCP만 있는 것이 아닙니다." 실시간 스트리밍이나 온라인 게임에서는 TCP 대신 **UDP(User Datagram Protocol)**를 사용합니다. 이때도 통신은 **UDP/IP** 형태로 이루어집니다. 즉, IP는 모든 인터넷 통신의 공통 분모이며, 그 위에 TCP라는 신뢰성 엔진을 얹었을 때 비로소 우리가 아는 안정적인 웹 브라우징과 파일 전송이 가능해집니다.

> **Key Insight**
> 우리가 'TCP/IP'라고 부르는 이유는, 데이터의 **목적지를 찾는 엔진(IP)**과 데이터의 **내용을 보증하는 엔진(TCP)**이 결합되어야만 비로소 완전한 통신 체계가 완성되기 때문입니다.
